Таблица 2: Заголовки

Содержат дополнительную информацию о запросе или ответе

Компонент HTTP-запрос (Request) HTTP-ответ (Response)
Описание Содержат метаинформацию о запросе, например, тип данных, cookies, информация о клиенте. Содержат метаинформацию об ответе, например, тип данных, сервер, cookies и т.д.
Обязательные данные - Заголовок Host (обязателен для HTTP/1.1) Нет строго обязательных заголовков
Примеры Обязательные заголовки:
- Host: www.example.com
- Content-Type: application/json (если тело запроса — JSON).
Необязательные заголовки:
- User-Agent: Mozilla/5.0
- Accept: text/html
- Authorization: Bearer token
- Cookie: sessionid=abc123 		Часто используемые заголовки:
- Content-Type: text/html
- Content-Length: 1234
- Server: Apache/2.4.1
- Set-Cookie: sessionid=12345
- Cache-Control: no-cache
Особенности в fetch Заголовки задаются в объекте headers.
headers: {
'Content-Type': 'application/json',
'Authorization': 'Bearer token'
} 		Заголовки ответа доступны через объект Response. response.headers.get('Content-Type')

Таблица 3: Тело

Передает данные, если они есть

Компонент HTTP-запрос (Request) HTTP-ответ (Response)
Описание Содержит данные, отправляемые на сервер (например, JSON, данные формы). Содержит данные, возвращаемые сервером (например, HTML, JSON, файл).
Обязательные данные Тело обязательно для методов POST, PUT, PATCH. Для GET и DELETE тело обычно отсутствует. Тело не обязательно для кодов состояния 204 No Content и 304 Not Modified. Для 200 OK тело обычно присутствует.
Примеры JSON:
body: JSON.stringify({"username": "user123",
"password": "pass123"})
Форма:
username=user123&password=pass123 		HTML:
<html>
<head><title>Example Page</title></head>
<body><h1>Hello, World!</h1></body>
</html>
JSON:
{"status": "success", "data": {"id": 1, "name": "John Doe"}}
Особенности в fetch body: JSON.stringify({ key: 'value' })
Для GET и DELETE тело не используется. 		response.json().then(data => console.log(data))
